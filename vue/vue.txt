  - src/components/CarSearchForm.vue ‚ÄĒ inputs for make (required), optional year

  - src/components/CarTable.vue ‚ÄĒ renders rows (id, make, model, year); clicking

  - src/components/CarDetailModal.vue ‚ÄĒ a simple modal showing the full
  CarDetail, including the image_url. Fetches on open, has its own loading/error
  state.
  - src/views/CarsView.vue ‚ÄĒ orchestrates: holds search params, pagination
  (limit, offset), the list, selected-car state, and the three UI states (empty,
  loading, error). Registered as the / route.

  Data flow ‚ÄĒ Empty state on load with a search bar. On search ‚Üí searchCars({ 
  make, year?, model?, limit, offset }) fills the table. Click a row ‚Üí open
  modal ‚Üí getCar(id) ‚Üí show details. Prev/Next adjust offset by limit and
  re-fetch. (Note: the list endpoint returns a bare array with no total count,
  so Next is enabled while a full page comes back and disabled when a
  short/empty page returns.)

  Error handling ‚ÄĒ 422 validation errors surface field messages near the form;
